## Описание проекта

**AI Client Report Generator** — это учебный Python-проект для автоматической генерации PDF-отчётов с помощью ИИ.

Проект принимает текстовую транскрибацию диалога с клиентом или данные о товаре, отправляет данные в OpenAI, получает структурированный результат, подставляет его в HTML-шаблон и сохраняет итоговый PDF-файл.

Также проект поддерживает работу через Telegram-бота.

---

## Цель проекта

Цель проекта — автоматизировать создание нескольких типов PDF-документов с использованием:
- обработки текста через ИИ;
- генерации структурированного JSON;
- HTML-шаблонов;
- генерации изображений;
- конвертации HTML в PDF;
- Telegram-бота для удобного взаимодействия.

---

## Основные возможности

Проект поддерживает 3 сценария генерации PDF:

### 1. Базовый отчёт по диалогу
Из текстовой транскрибации диалога создаётся PDF-отчёт с основной информацией о клиенте и его запросе.

В отчёт могут входить:
- имя клиента;
- компания;
- должность / роль;
- тема разговора;
- основной запрос;
- бизнес-цель;
- бюджет;
- сроки;
- целевая аудитория;
- боли клиента;
- обязательные пожелания;
- настроение;
- следующие шаги.

### 2. Отчёт по заказу для генерации дизайна сайта
Из транскрибации диалога система:
- извлекает требования к сайту;
- формирует prompt для генерации изображения;
- генерирует пример главной страницы сайта;
- вставляет изображение в PDF-отчёт.

Этот сценарий нужен для демонстрации того, как ИИ может превращать текстовые требования клиента в визуальный пример страницы сайта.

### 3. Карточка товара для маркетплейса
Пользователь передаёт:
- название товара;
- стоимость.

После этого система:
- генерирует описание товара;
- создаёт prompt для изображения;
- генерирует изображение товара;
- формирует PDF-карточку товара с:
  - изображением,
  - названием,
  - ценой,
  - описанием.

## Сценарии работы системы

В проекте реализованы разные режимы ответа системы.

### Сценарий 1 — базовый отчёт
**Вход:** `.txt` файл с транскрибацией  
**Выход:** PDF-отчёт по клиентскому диалогу

### Сценарий 2 — дизайн сайта
**Вход:** `.txt` файл с транскрибацией  
**Выход:** PDF-отчёт с описанием требований и изображением примера главной страницы сайта

### Сценарий 3 — карточка товара
**Вход:** название товара и цена  
**Выход:** PDF-карточка товара с изображением, названием, ценой и описанием

Таким образом, проект демонстрирует изменение логики работы в зависимости от выбранного сценария.
